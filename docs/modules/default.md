# Default Module

The Default module is the core rendering module that handles all standard markdown elements and HTML page structure. It must be included as it provides fallback rendering for any tokens not handled by other modules.

## Options

All options for the default module can be placed directly in the module configuration block without a specific `[default]` section header, or explicitly under `[default]`.

### Options Inheritance

Child pages automatically inherit all Default module options from their parent page. Options specified in the child overwrite inherited values. Set `root = true` to disable inheritance and start fresh.

**Example:**

```markdown
<!-- index.md: Sets site-wide defaults -->
<!--
title = "My Site"
[meta]
author = "Your Name"
-->

<!-- about.md: Inherits author, only changes title -->
<!--
title = "About"
-->

<!-- blog/index.md: Starts fresh, no inheritance -->
<!--
root = true
title = "Blog"
-->
```

For objects like `meta`, child values merge with parent values. Child keys override parent keys.
### Page Options

#### `root`
- **Type:** Boolean
- **Default:** `false`
- **Description:** If true, this page won't inherit options from its parent page. Use this to start a new section with different settings.

#### `title`
- **Type:** String
- **Optional**
- **Description:** Sets the HTML page title. If not specified, the first H1 heading in the markdown will be used. The title appears in the browser tab and is used for SEO.

#### `language`
- **Type:** String
- **Default:** `"en"`
- **Description:** Sets the language attribute for the HTML document. Use standard language codes (e.g., "en", "de", "fr", "es").

#### `meta`
- **Type:** Object (key-value pairs)
- **Default:** `{}`
- **Description:** Adds custom meta tags to the HTML head. Keys become meta tag names, values become the content.

**Example:**
```toml
[meta]
description = "My awesome blog post"
keywords = "rust, static site generator, markdown"
author = "Your Name"
```

#### `head`
- **Type:** Array of Strings
- **Default:** `[]`
- **Description:** Inject custom HTML elements into the `<head>` section. Useful for adding custom scripts, styles, or other head elements.

**Example:**
```toml
head = [
    '<script src="https://example.com/analytics.js"></script>',
    '<link rel="preconnect" href="https://fonts.googleapis.com">'
]
```

### Footer Options

Footer configuration is nested under the `footer` key.

#### `footer.items`
- **Type:** Array of Strings
- **Default:** `[]`
- **Description:** List of HTML elements or text to include in the footer.

#### `footer.custom`
- **Type:** String (HTML)
- **Optional**
- **Description:** Completely replaces the default footer with custom HTML. When set, `items` and `watermark` are ignored.

#### `footer.watermark`
- **Type:** Boolean
- **Default:** `true`
- **Description:** When true, adds "Generated by Lssg" to the footer.

**Example:**
```toml
[footer]
items = [
    '<p>© 2025 Your Name</p>',
    '<a href="/privacy">Privacy Policy</a>'
]
watermark = false
```

## Features

### Automatic Features

The Default module automatically handles:

- **Code Syntax Highlighting:** Uses Prism.js for syntax highlighting in code blocks
- **Favicon Support:** Automatically links `favicon.ico` if present in the site tree
- **Stylesheet Linking:** Automatically includes all CSS files in the correct order
- **Meta Tags:** Generates Open Graph and Twitter card meta tags for better social sharing
- **Responsive Viewport:** Adds responsive viewport meta tag
- **Script Injection:** Automatically includes JavaScript files at the end of the body

### Markdown Rendering

The Default module renders all standard markdown elements:
- Headings (H1-H6)
- Paragraphs
- Links
- Images
- Code blocks and inline code
- Blockquotes
- Lists (ordered and unordered)
- Emphasis (bold, italic)
- Horizontal rules
- HTML passthrough

```markdown
title = "My Page Title"

    '<p>© 2025 My Website</p>',
    '<a href="mailto:contact@example.com">Contact</a>'
]
watermark = true
-->

# My Page Content

Your markdown content here...
```
